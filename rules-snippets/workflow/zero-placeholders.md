## Workflow
- No placeholder comments like `// TODO: implement this` or `// rest of code` - every function must be fully implemented or not included
- If you cannot complete a function, explain why and ask for guidance - do not leave it half-done
- No stub returns like `return null` or `return {}` unless that is the correct behavior
- No commented-out code blocks - delete them, don't park them
- If a task is too large to complete in one pass, say so and break it into smaller tasks
